# agent-skills
privateなリポジトリから、共有可能なskillsをコピーしてきたリポジトリ

## 注意

ここにあるskillsは、Codex CLIを対象に調整されています。
他のコーディングエージェントでは意図通り動かないかもしれません。
調整した時期はものによって異なりますが、GPT-5.2〜5.4あたりです。

一部のスキルは、resources ディレクトリにあるファイルをシンボリックリンクで参照しています。
実際に利用する場合は`cp -L`などで、シンボリックリンクを解決してコピーしてください。

## スキル

### コードレビュー

#### codex/skills/pr-review

現在のワークスペースにチェックアウトされているブランチに対応するGitHubのプルリクエストに対して、コードレビューを行います。

- ghコマンドが利用可能である必要があります
    - 対象PRの特定、関連課題の特定、PRのdiff取得に利用します
- 複数観点ごとに、サブエージェントを起動してレビューを行うため、トークンの消費量が結構大きいので注意してください

#### codex/skills/diff-review

現在のワークスペースのdiffに対して、コードレビューを行います。
基本的な動作は、`pr-review`スキルと同様ですが、レビュー対象がプルリクエストのdiffではなく、現在のワークスペースのdiffになります。

`/goal`の機能とともに使うことを想定して、作成したものです。
GitHubの課題を与えると、それを要件として、要件の充足をチェックします。


### 開発ワークフロー

1. preparing-plan-for-issueを繰り返して情報をissueにためる
2. gen-exec-plan でExecPlanを生成する
3. exec-plans でExecPlanを実行する

というワークフローを想定したスキル群です。
ただ、`/goal`の機能が実装されたため、今後このワークフローを使うことは無い気がしています。
`preparing-plan-for-issue`でissueに必要な情報をためて、以下のような`goal`を与える形のほうが良いのではないかと思っています（が、これを書いている 2026/06/18時点では、大きめな実装では試していません）

```
/goal 以下の状態を全て満たすようにしてください

- 課題 ${URL} の本文及びコメントに書かれた要件・決定事項を全て満たす状態にしてください。
- コードレビューと修正を繰り返し、重要な指摘が無い状態にしてください
    - 実装が終わったら、`diff-review`スキルを用いてコードレビューを行う
    - レビュー結果を検証し、正しい指摘をすべて修正する
    - 修正が終わったら、新規にコードレビューと修正を行い、正しい指摘が無くなるまで繰り返す
```


#### codex/skills/preparing-plan-for-issue

指定されたGitHubのissueに対して、ExecPlanを作成するための事前情報を決定し、コメントとして情報を追加します。

- ghコマンドが必要です
- plan modeで開始することを想定しています（選択肢の提示と入力UIが便利なので）
- 過去の利用では、コメント数が20を超えてくると、すでに決めた情報を再度決めようとすることが多かったので、要修正だと思っています


#### codex/skills/gen-exec-plan

指定されたissueを元にExecPlanを作成します。

- ghコマンドが必要です


#### codex/skills/exec-plans

生成したExecPlanを用いて実際にコーディングを行います。
