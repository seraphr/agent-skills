## 前提

本スキルは、実装対象のGitHub課題が指定されていることを前提としています。
ユーザの指示に、GitHub課題の指定が無い場合は、ユーザに対して指定するように促してください。

## 手順

以下の手順は、`plan tool`（TODOs）を用いて実施してください。
この手順は非常に長く、あなたが手順の詳細を忘れてしまう可能性があります。
忘れてしまうのを防ぐため、各項目は、要約せず、サブアイテムを含めて全文をTODOに入れてください。

一度TODOに入れたアイテムを削除・要約・編集することは禁止します。
途中に新しいアイテムを追加することは許可しますが、その際は、既存のアイテムを削除・要約・編集せずに、新しいアイテムを追加してください。

1. ExecPlansについて理解する
    - `./references/PLANS.md`の内容を確認して、ExecPlanの構造、目的、使用方法を理解する
2. `<skill root>/scripts/show_issue.sh`を実行して、課題の内容を確認する
    - 課題の説明、目的、背景を把握する
    - コメントの内容をすべて確認し、要件・仕様・決定事項を理解する
3. 足りない情報を収集する
    - 課題とコメントの内容を分析して、ExecPlanを作成するために足りない情報がないかを分析する
    - 足りない情報がある場合は、ユーザーに質問して収集する
    - 足りない情報がなくなるまで、必要に応じて質問を繰り返す
4. ExecPlanを作成する
    - 初期ExecPlanを作成し、指定されたディレクトリにファイルとして出力する
    - 課題に記載の決定事項、仕様などは全ての内容を省略せずにExecPlanに反映する
        - 課題に記載の決定事項は、外すことのできない必須の要件で、ExecPlanの実行中に変更することは許されません
        - ExecPlanには、それらの決定事項が変更不可能な必須の要件であることを明記してください
    - 出力先が指定されていない場合は`.agent/plans/`ディレクトリに出力する
5. ExecPlanの**新規サブエージェントを用いた**検証と修正を指摘が無くなるまで繰り返す
    - 注意: 過去の記憶の無い状態でレビューを行わせるため、必ず新規のサブエージェントを起動させてください
    - サブエージェントを起動し、作成したExecPlanを検証させる
        - 課題とコメントに記載の決定事項、仕様などの全てがExecPlanに反映されているか
            - それら決定事項・仕様が、変更不可能な必須の要件であることがExecPlanに明記されているか
        - 課題とコメントの内容とExecPlanの内容に矛盾がないか
        - ExecPlanの内容自体に不備や矛盾がないか
        - サブエージェントには、`show_issue.sh`のフルパスと使い方を伝えて、課題とコメントの内容を確認させるように指示すること
    - サブエージェントから報告を受け取る
    - 不備・矛盾が指摘された場合
        - 指摘内容が正しいかどうかを確認する
        - 指摘内容が正しい場合は、ExecPlanを修正する
    - 不備・矛盾が指摘されなくなるまで、5を繰り返す
6. ユーザーにExecPlanの出力先を報告する


## Available scripts

スキル配下には、以下のスクリプトが用意されています。

### `<skill root>/scripts/show_issue.sh`

指定された課題の内容を表示するコマンドです。
課題の本文、コメントを全て取得して標準出力に出力します。

実行例:

```bash
<skill root>/scripts/show_issue.sh <issue_url>
```
