## 前提

このスキルでは、指定された課題を解決するためのExecPlanを作成するために必要な情報を収集し、要件を決定し、意思決定を行い、方式設計を行います。
ExecPlanを作成するための事前情報は、順次指定の課題にコメントとして追記していきます。

## 手順

1. ExecPlansについて理解する
    - `references/PLANS.md`の内容を確認して、ExecPlansの構造、目的、使用方法を理解し、どのような情報が必要かを把握する
2. `<skill root>/scripts/show_issue.sh`を実行して、課題の内容を確認する
    - 課題の説明、目的、背景、現在までに決まっていることを把握する
        - 特にコメントのタイトルを確認することで、すでに決まっている内容の種類を把握できる
    - どのような成果物が求められているかを明確にする
    - 必要であれば関連課題や関連の既存実装なども確認して、分析に役立てる
3. 新しい事前情報の候補をいくつかユーザーに提示して選択させる
    - 具体的にユーザから決めたい内容が指定されている場合は、選択をスキップして次に進む
4. 選択された事前情報について、ユーザーに質問し、判断をさせる
    - 曖昧な部分がなくなるまで、必要に応じて質問を繰り返す
5. 回答結果が、既存の事前情報と矛盾していないかを確認し、矛盾していたら矛盾している内容をユーザーに提示して、矛盾を解消するための判断をさせる
6. 結果をissueのコメント案の形にまとめてユーザに提示し、コメントを追加して良いかどうかを質問する
7. ユーザの承認を得るまで、必要に応じて情報を修正する
8. 結果をコメントとして追加する
    - `gh issue comment`コマンドを利用してコメントを追加する
        - コメント内容はファイルを経由せず直接引数に指定する

## コメントの書式

コメントのスケルトンは以下の通り

```md
## ${内容を端的に表すタイトル}

タイトルは決定事項の種類を表すものとする。

- 例
    - `API の具体契約に関する事前設計`
    - `編集画面の未保存変更ポリシーに関する事前設計`
    - `表示画面の表示設定APIとイベント取得経路`
    - `viewer の表示設定保存UIと 障害時挙動`
    - `表示における既存コード再利用方針`

### 種別

情報の種別を記述する。

- 例: 要件、仕様、決定事項、方式設計、実証コード など

#### ${内容を端的に表すタイトル1}

- 情報の具体的な内容を記述する。
- 情報には具体的な方針・仕様に加えて、その理由や背景、選択肢の中からなぜそれを選んだのかなどの判断の根拠も含めるとより良い。


#### ${内容を端的に表すタイトル2}

- 1コメントが複数の決定事項で表される場合は、複数のタイトルをつけても良い
- 個別の決定事項ごとに、理由や背景を記述する

```

## Available scripts

スキル配下には、以下のスクリプトが用意されています。

### `<skill root>/scripts/show_issue.sh`

指定された課題の内容を表示するコマンドです。
課題の本文、コメントを全て取得して標準出力に出力します。

実行例:

```bash
<skill root>/scripts/show_issue.sh <issue_url>
```
