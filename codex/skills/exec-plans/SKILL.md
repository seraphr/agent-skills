## 前提

本スキルは、実装対象のGitHub課題と初期のExecPlanが指定されていることを前提としています。
ユーザの指示に、それらの指定が無い場合は、ユーザに対して指定するように促してください。


## 作業中の重要な注意事項

- `ExecPlansを用いた実装の手順` は `plan tool`（TODOs）を用いて実施してください。
    - この手順は非常に長く、あなたが手順の詳細を忘れてしまう可能性があります。
    - 忘れてしまうのを防ぐため、各項目は、要約せず、サブアイテムを含めて全文をTODOに入れてください。
- `exec-plans` スキルに書いてある手順を転記した**plan tool（TODOs）のアイテムを削除・変更することは禁止**です
    - 新しいタスクを追加することはしても良いですが、既存のタスクの内容を一字一句変更してはなりません
- タスクが一つ終わり度に、 `exec-plans` スキルの内容を再度読み込み、正確な実施手順を思い出してください
    - その際に、plan toolに登録されている内容が、スキルの手順と異なっているなら、手順を正確に登録しなおしてください
 

## ExecPlansを用いた実装の手順

1. ExecPlansについて理解する
    - `./references/PLANS.md`の内容を確認して、ExecPlanの構造、目的、使用方法を理解する
2. 指定されたExecPlanファイルを用いて最後まで実装を行う
    - 指定されたGitHub課題の本文及び各コメントも`ソース資料`として取り扱い、丁寧に読み、必要になったら再読してください。
3. 実装結果の**新規サブエージェントを用いた**検証と修正を指摘が無くなるまで繰り返す
    - 注意: 過去の記憶の無い状態でレビューを行わせるため、必ず新規のサブエージェントを起動させてください
    - サブエージェントを起動し、実装結果を検証させる
        - 注意: サブエージェントにはExecPlanの情報は渡さず、課題だけを見せること
        - 課題とコメントの全文を確認させる
            - 課題本文の取得: `gh issue view "${issue_url}"`
            - 課題コメントの取得: `gh issue view "${issue_url}" --comments`
        - 課題に記載の決定事項、仕様などの全てが実装に反映されているか
        - 課題の内容と実装の内容に矛盾がないか
        - 実装の内容自体に不備や矛盾がないか
    - サブエージェントの報告を受け取る
    - 不備・矛盾が指摘された場合
        - 指摘内容が正しいかどうかを確認する
        - 指摘内容が正しい場合は、実装を修正する
    - 不備・矛盾が指摘されなくなるまで、3を繰り返す
4. ユーザーに実装の完了を報告する
